<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → PublicCommentController | Untrusted input (authorName, content) via POST /api/articles/{id}/comments |
| client → CommentSection.vue | Same untrusted input flows to API |
| PublicCommentController → CommentService | Validated input passes through |
| CommentService → CommentMapper | MyBatis #{ } binding prevents SQL injection |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-20 | XSS | CommentSection.vue | mitigate | All user content rendered via Vue `{{ }}` text interpolation — HTML escaped automatically. No v-html on comment content. |
| T-03-21 | SQL Injection | commentApi.submit → backend | mitigate | Backend uses MyBatis `#{ }` for all user fields in CommentMapper.insert |
| T-03-22 | Denial of Service | CommentSection form | mitigate | Honeypot field catches bots; form validates non-empty before submit; success message shown even on errors to avoid bot feedback |
| T-03-23 | Spam | PublicCommentController | mitigate | Honeypot check in CommentService — filled honeypot throws SpamException, silently accepted |
</threat_model>
