```java
package com.blog.service.impl;

@Service
public class UserServiceImpl implements UserService {
    @Autowired
    private UserMapper userMapper;
    @Autowired
    private PasswordEncoder passwordEncoder;
    @Autowired
    private JwtUtils jwtUtils;

    @Override
    public String authenticate(String username, String password) {
        User user = userMapper.findByUsername(username);
        if (user == null) {
            return null;  // User not found
        }
        if (!passwordEncoder.matches(password, user.getPassword())) {
            return null;  // Wrong password
        }
        return jwtUtils.generateToken(username);
    }

    @Override
    public User findByUsername(String username) {
        return userMapper.findByUsername(username);
    }
}
```

Use BCryptPasswordEncoder for password hashing. Use JwtUtils for token generation.

Create JwtUtils.java for token generation and validation:

```java
package com.blog.config;

@Component
public class JwtUtils {
    @Value("${jwt.secret}")
    private String secret;

    @Value("${jwt.expiration}")
    private long expiration;

    public String generateToken(String username) {
        return Jwts.builder()
            .subject(username)
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + expiration))
            .signWith(Keys.hmacShaKeyFor(secret.getBytes()))
            .compact();
    }

    public String validateToken(String token) {
        try {
            return Jwts.parser()
                .verifyWith(Keys.hmacShaKeyFor(secret.getBytes()))
                .build()
                .parseSignedClaims(token)
                .getPayload()
                .getSubject();
        } catch (JwtException e) {
            return null;
        }
    }
}
```

Create JwtAuthenticationFilter.java as OncePerRequestFilter:
- Extract JWT from `Authorization: Bearer <token>` header
- Call `jwtUtils.validateToken(token)`
- If valid, set SecurityContext with UsernamePasswordAuthenticationToken
- If invalid/missing, continue filter chain (let SecurityConfig decide access)

**Important:** Do NOT throw exceptions on invalid tokens — just return null from validateToken and continue.

Create SecurityConfig.java:

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Autowired
    private JwtAuthenticationFilter jwtAuthFilter;
    @Autowired
    private RateLimitFilter rateLimitFilter;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)  // Disable CSRF for REST API
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))  // Allow frontend on :5173
            .sessionManagement(session -> session.sessionCreationPolicy(STATELESS))  // JWT stateless
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/login").permitAll()  // Login is public
                .requestMatchers("/api/auth/logout").permitAll()  // Logout is public
                .requestMatchers("/uploads/**").permitAll()       // Static files public
                .requestMatchers("/api/admin/**").authenticated()  // Admin endpoints require auth
                .anyRequest().permitAll()  // All other endpoints public
            )
            .addFilterBefore(rateLimitFilter, UsernamePasswordAuthenticationFilter.class)
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

Create RateLimitFilter.java as OncePerRequestFilter:

**Implementation:**
- Use ConcurrentHashMap to track login attempts per IP: `Map<String, LoginAttempt> attemptMap`
- LoginAttempt holds: count (int), firstAttemptTime (long)
- On `/api/auth/login` request: extract client IP from HttpServletRequest
- Check if IP has exceeded 5 attempts within 60 seconds
- If exceeded: set response status to 429 Too Many Requests, return immediately
- If not exceeded: increment count, let request continue
- After successful login: reset the counter for that IP

**Thread safety:** Use ConcurrentHashMap with compute() for atomic updates.

**Login path detection:** Check if request.getRequestURI() contains "/auth/login" and request.getMethod() is "POST".

**Default behavior:** If not a login request, just call filterChain.doFilter() immediately.
