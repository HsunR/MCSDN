**Interface** `backend/src/main/java/com/blog/service/ImageDownloadService.java`:

```java
package com.blog.service;

public interface ImageDownloadService {
    /**
     * Downloads and replaces CSDN image URLs in Markdown content.
     * Uses MD5 hash of URL for deduplication.
     * Retries failed downloads 3 times with exponential backoff (1s, 2s, 4s).
     * @param content Markdown content with CSDN image URLs
     * @return Content with replaced local paths; original URLs kept on failure
     */
    String downloadAndReplaceImages(String content);
}
```

**Implementation** `backend/src/main/java/com/blog/service/impl/ImageDownloadServiceImpl.java`:

Dependencies:
- `@Value("${upload.path:./uploads}") private String uploadPath;`
- `@Autowired private RestClient.Builder restClientBuilder;`
- `@Autowired private DownloadedImageMapper downloadedImageMapper;`

Key implementation details:
1. Regex pattern: `!\[([^\]]*)\]\((https?://[^)]+\.(?:jpg|jpeg|png|gif|webp)(?:\?[^\)]*)?)\)` — only match Markdown images with image extensions
2. Compute MD5 hash of URL (NOT content) using `MessageDigest.getInstance("MD5")` + `HexFormat.of().formatHex()`
3. Check `downloadedImageMapper.findByUrlHash(urlHash)` — if exists, return localPath directly (skip download)
4. If not exists: download via `restClientBuilder.build().get().uri(url).retrieve().body(byte[].class)` with 10s connect timeout, 30s read timeout
5. Retry 3 times with `Thread.sleep((long) Math.pow(2, attempt) * 1000)` between attempts (1s, 2s, 4s)
6. Infer extension from Content-Type header or URL (default to "jpg")
7. Save to `{uploadPath}/{year}/{month}/{urlHash}.{ext}` using `Files.write()`
8. Insert into `downloaded_images` table with urlHash, localPath, originalUrl
9. Return new local path: `/uploads/{year}/{month}/{urlHash}.{ext}`
10. On all retries exhausted, log error and return `null` (caller keeps original URL)
11. Only process `http` and `https` URLs — reject anything else (SSRF mitigation)

Create directories with `Files.createDirectories()` before writing. Use `Paths.get()` for path construction.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | S | ImageDownloadServiceImpl | mitigate | Validate URL scheme is http or https only; reject file://, gopher://, etc. to prevent SSRF |
| T-05-02 | T | ImageDownloadServiceImpl | mitigate | Use `Paths.get(uploadPath, year, month, filename)` — path is constructed from fixed parts, filename is UUID-equivalent hash, not user input |
| T-05-03 | D | DownloadedImageMapper | mitigate | urlHash is computed server-side via MD5 of URL string, not user-supplied; SQL uses parameterized #{urlHash} in findByUrlHash |
| T-05-04 | I | RestClient download | accept | External image content is written to disk with server-controlled extension; Content-Type header determines actual format |
| T-05-05 | R | Image download | mitigate | Configure RestClient with setConnectTimeout(10_000) and setReadTimeout(30_000) to prevent resource exhaustion |

**Security controls implemented:**
- URL allowlist: only `http://` and `https://` schemes accepted (T-05-01)
- Path sanitization: filename is MD5 hash (not user-supplied), path constructed from fixed upload.path config (T-05-02)
- SQL injection: MyBatis parameterized queries (#{urlHash}) prevent injection (T-05-03)
- Timeout: RestClient configured with explicit timeouts (T-05-05)
</threat_model>
