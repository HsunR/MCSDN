Modify `backend/src/main/java/com/blog/service/impl/CsdnSyncServiceImpl.java`:

1. Add new field:
```java
@Autowired
private ImageDownloadService imageDownloadService;
```

2. In `syncArticles()`, after line where `CsdnArticleDto articleDto = csdnArticleParser.parseArticle(html, articleUrl);` (around line 72), add:

```java
// Replace CSDN image URLs with local paths (SYNC-07, SYNC-09)
try {
    String contentWithLocalImages = imageDownloadService.downloadAndReplaceImages(articleDto.getContent());
    articleDto.setContent(contentWithLocalImages);
} catch (Exception e) {
    // Image download failed — continue with original content per continue-on-error pattern
    log.warn("Image download failed for article {}: {}", articleUrl, e.getMessage());
}
```

The try/catch wraps only the image download call — article sync continues regardless of image download result.

3. When inserting new article (line ~90), the article content already has local paths from step 2.

4. When updating existing article (line ~97), the article content also has local paths from step 2 — images that were already downloaded will be skipped by MD5 dedup in ImageDownloadService.
