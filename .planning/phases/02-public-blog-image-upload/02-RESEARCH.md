# Phase 2: Public Blog & Image Upload - Research

**Researched:** 2026-04-13
**Domain:** Public blog frontend (Vue 3) + Admin category/tag management (Spring Boot) + Image upload (Spring Boot + local filesystem)
**Confidence:** MEDIUM

## Summary

Phase 2 builds the public-facing blog (article list with timeline layout, article detail, category/tag filtering, search) and adds admin category/tag CRUD plus image upload functionality. The existing Spring Boot backend uses MyBatis 3.0.4 with MySQL; the Vue 3 frontend uses Vue Router hash mode with Pinia stores. Key implementation decisions: MySQL FULLTEXT for search (not Elasticsearch), local filesystem for image storage, slug auto-generated from title, excerpt auto-stripped from markdown content.

**Primary recommendation:** Use MyBatis XML mappers for paginated article queries (LIMIT/OFFSET), add `WebMvcConfigurer` for Spring Boot static resource serving of `/uploads/**`, use a simple Java slugify utility (no external library needed), and use `striptags` npm package on frontend for excerpt generation.

### Pattern 1: Public Article List with Pagination and Filters

**Backend MyBatis XML (ArticleMapper.xml):**

```xml
<!-- Public article list with pagination -->
<select id="findPublishedPaginated" resultMap="ArticleResultMap">
  SELECT a.id, a.title, a.content, a.status, a.category_id, a.created_at, a.updated_at,
         c.name as category_name, c.id as cat_id
  FROM articles a
  LEFT JOIN categories c ON a.category_id = c.id
  WHERE a.status = 'PUBLISHED'
  ORDER BY a.created_at DESC
  LIMIT #{limit} OFFSET #{offset}
</select>

<!-- Count for pagination -->
<select id="countPublished" resultType="int">
  SELECT COUNT(*) FROM articles WHERE status = 'PUBLISHED'
</select>

<!-- Filter by category slug -->
<select id="findByCategorySlug" resultMap="ArticleResultMap">
  SELECT a.id, a.title, a.content, a.status, a.category_id, a.created_at, a.updated_at
  FROM articles a
  INNER JOIN categories c ON a.category_id = c.id
  WHERE a.status = 'PUBLISHED' AND c.name = #{categorySlug}
  ORDER BY a.created_at DESC
  LIMIT #{limit} OFFSET #{offset}
</select>

<!-- Filter by tag slug -->
<select id="findByTagSlug" resultMap="ArticleResultMap">
  SELECT DISTINCT a.id, a.title, a.content, a.status, a.category_id, a.created_at, a.updated_at
  FROM articles a
  INNER JOIN article_tags at ON a.id = at.article_id
  INNER JOIN tags t ON at.tag_id = t.id
  WHERE a.status = 'PUBLISHED' AND t.name = #{tagSlug}
  ORDER BY a.created_at DESC
  LIMIT #{limit} OFFSET #{offset}
</select>

<!-- MySQL FULLTEXT search -->
<select id="searchPublished" resultMap="ArticleResultMap">
  SELECT a.id, a.title, a.content, a.status, a.category_id, a.created_at, a.updated_at,
         MATCH(a.title, a.content) AGAINST(#{keyword} IN NATURAL LANGUAGE MODE) AS relevance
  FROM articles a
  WHERE a.status = 'PUBLISHED'
    AND MATCH(a.title, a.content) AGAINST(#{keyword} IN NATURAL LANGUAGE MODE)
  ORDER BY relevance DESC
  LIMIT #{limit} OFFSET #{offset}
</select>
```

**Source:** [MySQL 8.0 FULLTEXT Documentation](https://dev.mysql.com/doc/refman/8.0/en/fulltext-search.html) - Verified MATCH() AGAINST() syntax and LIMIT/OFFSET pagination.

### Pattern 3: Timeline Layout with Date Dividers (Vue 3)

```vue
<script setup>
import { computed } from 'vue'
import ArticleCard from './ArticleCard.vue'

const props = defineProps({
  articles: Array  // { id, title, excerpt, tags, date, category }
})

// Group articles by date string
const groupedByDate = computed(() => {
  const groups = {}
  for (const article of props.articles) {
    const dateKey = new Date(article.date).toLocaleDateString('zh-CN', {
      year: 'numeric', month: 'long', day: 'numeric'
    })
    if (!groups[dateKey]) groups[dateKey] = []
    groups[dateKey].push(article)
  }
  return groups
})
</script>

<template>
  <div class="timeline">
    <div v-for="(articles, date) in groupedByDate" :key="date" class="date-group">
      <div class="date-divider">
        <span class="date-label">{{ date }}</span>
      </div>
      <div class="articles">
        <ArticleCard
          v-for="article in articles"
          :key="article.id"
          :article="article"
        />
      </div>
    </div>
  </div>
</template>
```

### Pattern 4: Excerpt Generation (Strip Markdown + Truncate)

**Frontend (striptags):**
```javascript
import striptags from 'striptags'

function generateExcerpt(markdownContent, maxLength = 150) {
  // Strip markdown syntax, HTML tags, and excessive whitespace
  const plainText = striptags(markdownContent)
    .replace(/[#*`_~\[\]()]/g, '')  // remove markdown symbols
    .replace(/\s+/g, ' ')           // normalize whitespace
    .trim()

  if (plainText.length <= maxLength) return plainText
  return plainText.substring(0, maxLength).replace(/\s+\S*$/, '') + '...'
}
```

**Source:** [VERIFIED: npm registry - striptags 3.2.0](https://www.npmjs.com/package/striptags)

### Pattern 5: Spring Boot Image Upload

**Controller:**
```java
package com.blog.controller;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.multipart.MultipartFile;

import java.io.IOException;
import java.nio.file.Files;
import java.nio.file.Path;
import java.nio.file.Paths;
import java.time.LocalDate;
import java.util.UUID;

@RestController
@RequestMapping("/api/admin/upload")
public class UploadController {

    @Value("${upload.path:./uploads}")
    private String uploadPath;

    private static final Set<String> ALLOWED_TYPES = Set.of(
        "image/jpeg", "image/png", "image/gif", "image/webp"
    );

    @PostMapping
    public ResponseEntity<Map<String, String>> uploadImage(
            @RequestParam("file") MultipartFile file) {

        // Validate content type
        if (!ALLOWED_TYPES.contains(file.getContentType())) {
            return ResponseEntity.badRequest()
                .body(Map.of("error", "Invalid file type"));
        }

        // Validate size (already enforced by spring.servlet.multipart.max-file-size=5MB)

        // Generate path: /uploads/{year}/{month}/{uuid}.{ext}
        LocalDate now = LocalDate.now();
        String ext = getExtension(file.getOriginalFilename());
        String filename = UUID.randomUUID() + "." + ext;
        Path yearMonthDir = Paths.get(uploadPath,
            String.valueOf(now.getYear()),
            String.format("%02d", now.getMonthValue()));
        Path filePath = yearMonthDir.resolve(filename);

        // Create directories and write file
        Files.createDirectories(yearMonthDir);
        file.transferTo(filePath);

        // Return public URL path
        String url = "/uploads/" + now.getYear() + "/" +
            String.format("%02d", now.getMonthValue()) + "/" + filename;

        return ResponseEntity.ok(Map.of("url", url));
    }
}
```

**Source:** [Spring Framework WebMvcConfigurer static resources](https://docs.spring.io/spring-framework/reference/web/webmvc/mvc-config/static-resources.html) - Verified `addResourceHandler` + `addResourceLocations` pattern.

### Pattern 6: Static Resource Serving for /uploads/**

```java
package com.blog.config;

import org.springframework.beans.factory.annotation.Value;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.config.annotation.ResourceHandlerRegistry;
import org.springframework.web.servlet.config.annotation.WebMvcConfigurer;

@Configuration
public class WebConfig implements WebMvcConfigurer {

    @Value("${upload.path:./uploads}")
    private String uploadPath;

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/uploads/**")
                .addResourceLocations("file:" + uploadPath + "/");
    }
}
```

**Important:** SecurityConfig already has `.requestMatchers("/uploads/**").permitAll()` so unauthenticated access works.

**Source:** [Spring Framework WebMvcConfigurer](https://docs.spring.io/spring-framework/reference/web/webmvc/mvc-config/static-resources.html)

### Pattern 7: Java Slugify Utility

```java
package com.blog.util;

public class SlugUtils {
    public static String slugify(String input) {
        if (input == null || input.isBlank()) return "";

        return input
            .toLowerCase()
            .trim()
            .replaceAll("[^a-z0-9\\s-]", "")  // remove special chars
            .replaceAll("\\s+", "-")           // spaces to hyphens
            .replaceAll("-+", "-")            // collapse multiple hyphens
            .replaceAll("^-|-$", "");          // trim leading/trailing hyphens
    }
}
```

**Why not use a library:** Apache Commons Text has `StringEscapeUtils.escapeHtml4()` which is for HTML, not URL slugification. A custom utility is ~5 lines and has zero dependencies. [ASSUMED]

### Anti-Patterns to Avoid

- **Do NOT use LIKE %keyword% for search** - Does not leverage MySQL FULLTEXT index; poor performance on large datasets. Use `MATCH() AGAINST()` instead.
- **Do NOT store images as Base64 in database** - Database bloat, performance issues. Use local filesystem with path stored in article content.
- **Do NOT use session-based auth for public API** - The existing JWT approach is stateless; public blog endpoints should simply not require authentication.
- **Do NOT use Elasticsearch in Phase 2** - Added complexity and infrastructure requirements. MySQL FULLTEXT is sufficient for a personal blog.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Image storage path generation | Custom Path construction | Java `Paths.get()` + `UUID.randomUUID()` | Handles edge cases across OS |
| Markdown rendering | Custom parser | markdown-it (already installed) | Handles all edge cases |
| HTML escaping in excerpt | String replace hacks | Apache Commons Text or OWASP ESAPI | Security: prevents XSS |
| Slug generation | Third-party library | Simple regex utility | 5 lines, zero dependencies |

## Common Pitfalls

### Pitfall 1: MySQL FULLTEXT Minimum Word Length
**What goes wrong:** Words with 3 or fewer characters (e.g., "vue", "css", "sql") may not be indexed by default `ft_min_word_len=4`.
**Why it happens:** MySQL FULLTEXT index has configurable minimum word length.
**How to avoid:** Either accept the limitation (4+ char words only), or reconfigure MySQL `ft_min_word_len=3` and restart MySQL. For personal blog with typical article content, this is usually acceptable.
**Warning signs:** Search for "vue" returns no results but "vue3" does.

### Pitfall 2: Image Upload Path on Windows vs Linux
**What goes wrong:** Hardcoded `/uploads/` path works on Linux but breaks on Windows; relative `./uploads/` works differently depending on working directory.
**Why it happens:** Path separator differences and working directory issues.
**How to avoid:** Use `Path` API with `file:` prefix in resource handler AND store configured path in `application.yml` as `${upload.path:./uploads}`. Always use `Files.createDirectories()` before writing.
**Warning signs:** Images upload successfully but 404 when accessing `/uploads/**`.

### Pitfall 3: Pagination Page Number Validation
**What goes wrong:** Negative or zero page numbers cause MySQL syntax errors with LIMIT.
**Why it happens:** No validation on `page` query parameter.
**How to avoid:** Validate page >= 1 in controller/service; default to page 1 for null/invalid.
**Warning signs:** `limit 10 offset -10` causes MySQL error.

### Pitfall 4: Category/Tag Deletion with Existing Articles
**What goes wrong:** Deleting a category/tag that has associated articles violates foreign key constraint or leaves articles with null category/tag.
**Why it happens:** V2 migration has `ON DELETE SET NULL` for category_id, but no equivalent for tags.
**How to avoid:** Admin UI should warn before deletion; consider soft-delete for tags or require no articles before deletion.
**Warning signs:** Article shows "Uncategorized" after category deletion.

### Pitfall 5: Markdown Image URLs in Content vs Static Serving
**What goes wrong:** Image URL stored as `![alt](/uploads/2026/04/uuid.png)` but Spring Boot static handler not configured.
**Why it happens:** Static resource handler needs explicit configuration in Spring Boot 3.
**How to avoid:** Ensure `WebConfig` adds resource handler for `/uploads/**` AND `SecurityConfig` permits access.
**Warning signs:** Images show broken link icon in MarkdownEditor preview.

### Public Article API Response DTO

```java
package com.blog.dto;

public class PublicArticleResponse {
    private Long id;
    private String title;
    private String slug;          // auto-generated from title
    private String excerpt;        // first 150 chars, stripped markdown
    private String content;        // full markdown
    private LocalDateTime createdAt;
    private CategoryDTO category;
    private List<TagDTO> tags;

    // 150-char excerpt generation
    public static String generateExcerpt(String markdown) {
        String text = markdown
            .replaceAll("!\\[.*?\\]\\(.*?\\)", "")  // remove image markdown
            .replaceAll("\\[.*?\\]\\(.*?\\)", "")   // remove link markdown
            .replaceAll("[#*`_~>\\-\\s]+", " ")    // remove formatting chars
            .trim();
        if (text.length() <= 150) return text;
        return text.substring(0, 147) + "...";
    }

    // Slug auto-generation
    public static String generateSlug(String title) {
        return title.toLowerCase()
            .replaceAll("[^a-z0-9\\s-]", "")
            .replaceAll("\\s+", "-")
            .replaceAll("-+", "-");
    }
}
```
